    # 在使用 ruoyi-vue-pro boot版，上线至公司内部系统时，遇到 java.io.FileNotFoundException: /home/ruoyi/logs/sys-user.log (No such file or directory)

    分析：此错误为启动时日志输出的文件没有找到，输出的日志无法保存导致的

    方案：admin模块下找到配置文件 logback.xml 日志输出在根目录下的home下的ruoyi/logs

    当前登录的用户没有home权限

    修改文件输出路径至相对路径即可，文件夹无需手动创建，修改后运行jar包，即自动创建

     <!-- 原 日志存放路径 -->
	<property name="log.path" value="/home/ruoyi/logs" />
 <!-- 修改后的路径 日志存放路径 -->
	<property name="log.path" value="log/sys-info" />

    # Cause: java.sql.SQLException: 无效的列类型: 1111

    1.情景展示 使用mybatis给Oracle表插入数据的时候，报错信息如下：

    org.mybatis.spring.MyBatisSystemException: nested exception is org.apache.ibatis.type.TypeException: Could not set parameters for mapping: ParameterMapping{property='VERCODE', mode=IN, javaType=class java.lang.Object, jdbcType=null, numericScale=null, resultMapId='null', jdbcTypeName='null', expression='null'}. Cause: org.apache.ibatis.type.TypeException: Error setting null for parameter #6 with JdbcType OTHER . Try setting a different JdbcType for this parameter or a different jdbcTypeForNull configuration property. Cause: java.sql.SQLException: 无效的列类型: 1111

    2.具体分析 我们知道，在SQL语句当中，如果我们在执行插入的时候，指定了某个字段，但并没有给值的情况下；

    默认情况下，在插入数据时，该字段值为NULL（也就是没有给这个字段赋值）

    3.解决方案

    # MyBatis配置 yam 文件中mybatis增加如下配置
mybatis:
  configuration:
    jdbc-type-for-null: 'null'

    # ORA-01017: 用户名/口令无效; 登录被拒绝

    1.情景展示 代码操作本地oracle数据库时，忘记密码了

    2.具体分析 修改密码或者新建一个用户并赋权即可

    3.解决方案

    打开命令行 cmd 按照如下命令操作即可 重置用户system的密码为oracle

    C:\Users\15391>sqlplus / as sysdba

SQL*Plus: Release 19.0.0.0.0 - Production on 星期五 9月 15 15:30:15 2023
Version 19.3.0.0.0

Copyright (c) 1982, 2019, Oracle.  All rights reserved.


连接到:
Oracle Database 19c Enterprise Edition Release 19.0.0.0.0 - Production
Version 19.3.0.0.0

SQL> alter user system account unlock;

用户已更改。

SQL> alter user system identified by oracle;

用户已更改。

SQL>

    # dynamic-datasource can not find primary datasource

    1.因为之前mybatis使用了多源数据，引入了dynamic-datasource-spring-boot-starter依赖，导致报错

    删除掉dynamic-datasource-spring-boot-starter依赖，或者在application中设置为多源数据

    注意：修改pom文件后，刷新一下maven

    # Cron expression must consist of 6 fields (found 7 in "0 30 2 * * ? *")

    Spring定时任务的cron表达式只能包含6个字段，即使不支持支持年份

    去掉cron表达式中的最后一行即可

    # ORA-12514, TNS:listener does not currently know of service requested in conn

    我这边遇到的问题是db_name=orcl，orcl写错了修正后可以连接数据库了
