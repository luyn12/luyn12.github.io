# SpringBoot

SpringBoot注意事项

1）注意版本问题【2.1.8.RELEASE】、JDK版本【1.8】
2）注意Idea环境配置【已发IDEA配置文档】
3）注意代码位置，要在“引导类”所在的包下或者其子包下

3

概念：基于spring4.x的框架，快速使用spring框架构建项目的一种方式

优点：

1）自动装配：约定大于配置
2）起步依赖：maven依赖传递
3）非功能性增强：嵌入式服务器、健康监测、日志...

起步依赖

1）所有springboot项目都要继承spring-boot-starter-parent
	在spring-boot-starter-parent进行了大量的jar包版本的锁定，不需要我们手动解决jar包冲突问题
2）添加起步依赖 spring-boot-starter-xxx
	在起步依赖spring-boot-starter-web中，已经引入了该web功能所需的其他所有jar包
优点：1）管理jar包版本，防止版本冲突 2）简化jar包依赖管理；

自动装配

约定大于配置：已经设置了默认值，如果不配置就使用默认值，如果配置了采用配置的值
(原因：每个起步依赖都有自己的autoconfigure默认配置的jar包，所以不需要在去手动进行配置，都可以使用默认值)
优点：简化配置管理

获取配置文件属性值的方式

三种方式：
1）@Value     				//单个属性值
	使用场景：注入属性比较少

	1）配置文件
	2）静态常量类
	3）枚举类

自动装配

springboot项目不要随便导入起步依赖，项目启动时会根据起步依赖进行自动装配，
	如果有默认配置，可以自动装配成功；
	如果没有默认配置，自动装配就会报错，项目启动就会失败；
总结：springBoot项目不要随便导起步依赖

思路分析

1）创建项目 redis-spring-boot-starter(起步依赖项目：封装jar包依赖、版本)

@Import注解

@Import:创建对象，并把对象注入IOC容器，开启某些功能（@EnableXxx底层实现）
	四种方式：
		1）@Import(User.class)
		2）@Import(UserConfig.class)
		3）@Import(XxxxImportSelector.class)
			XxxxImportSelector需要实现ImportSelector接口
			注：spring框架内部，大量使用了该种方式，把bean注入spring容器
			步骤：
				1)从配置文件中加载好多类的全限定名，例如：com.kdplm.User、com.kdplm.Address.
				2)根据类的全限定名，排除重复、排除懒加载数据，根据条件，生成Bean
				3)把Bean注入spring容器
		4）@Import(XxxxImportBeanDefinitionRegistrar.class)
			XxxxImportBeanDefinitionRegistrar需要实现ImportBeanDefinitionRegistrar接口

SpringBoot监听接口

	CommandLineRunner、ApplicationRunner
		特点：
			spring容器创建后执行；
			不需要配置(只需要在类上添加@Component注解即可)，实现后项目启动直接执行

Springboot加载核心流程

springBoot
	1）加载一系列监听器
	2）环境准备
	3）读取配置文件，获取所有要创建的实体类的全限定名
	4）去重、排除懒加载的bean，得到最终的类的全限定名的集合
	5）使用Bean工厂+反射，生成Bean，完善Bean的属性
	6）返回spring容器
